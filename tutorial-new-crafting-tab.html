Tutorial: Adding New Crafting Category Tab

Made using RPG I&I v1.9
Tutorial Result Example Consumables Tab Example with 2 Sub-Categories
1. Open E_CraftingCategory and add new enumerator to it (your new tab name)
2. Open WB_CraftingList, duplicate one of the ButtonSwitch widgets, rename it to ButtonConsumables, change the Category to Consumables. Also change the Button Style OFF/ON
3. Duplicate one of the MainVBoxes, rename it to MainVBox_Consumables
4. Go to Set List Title Text, set Text that will be displayed for your new Tab
5. Go to Switch Crafting Tab (this function is called only on Gamepad Trigger Left/Right Input), add/adjust the highlighted nodes with appropriate Category input param
6. Go to Get List by Category, connect your newly created VBox to the appropriate pin
7. Go to Get List Button, connect newly created Category Switch Button to the appropriate pin
8. Go to Event Graph, override the ButtonConsumables / On Clicked Event (select the Button variable -> OnClicked), connect Switch List node to it
9. On Event Switch List, pin ButtonConsumables to the Refresh List Button node
10. Open E_CraftingSubcategory (only if you want to add some special sub-categories to the List), and add new Enumerators, in this example Potions & Food
11. Open AC_CraftingCore / Filter by Rarity, if you want to use the Items with some other rarity than currently connected you will need to add it in there. In this example adding Consumables (new local variable + appending it at the beggining)
12. Duplicate one of the Crafting List Data Tables, add your new Crafting Datas to it - in this example adding Healing Potion to Category = Consumables
13. Open your PlayerController, select Player Crafting Component (if you want to be able to access your newly created tab in Player Crafting Window) add new Crafting Data Table to the Array
14. Open AC_CraftingCore (if you want to be able to access your newly created tab from every Crafting Station you interact with) add Crafting Data Table to the Array
